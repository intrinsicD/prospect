## Exact engineering cause

The accepted development-audit terminal's `primary` object correctly records:

```text
reproduction_runtime_file = development-audit-runtime-085a40783f42dc5c.json
```

`create_audit_reproduction_receipt()` deliberately writes this
content-addressed sidecar from the second captured audit execution. The
reproduction receipt names it, the terminal copies that exact name, the file is
present in the terminal manifest, and the central operator verifier plus the
independent formal-input auditor join the reference to its bytes.

The standard-library outer launcher's separate
`_verify_development_audit()` implementation instead requires the field to be
the hardcoded execution-capture name:

```python
primary.get("reproduction_runtime_file") == "audit-execution-02.runtime.json"
```

Both files exist and contain the same 2,039 bytes with SHA-256
`085a40783f42dc5c5f8b9ead0cffece1aa159a401e6450d0ee517a3fabfce1c8`.
The failure is therefore not a changed runtime manifest or missing audit
evidence. It is a stale reference-name expectation in the independent
standard-library consumer. Every other predicate in its combined terminal
shape check matches the accepted package. That bounded comparison does not by
itself establish that correcting this literal would make every later rehearsal
step pass.

The traceback occurs while `_open_typed_runtime_custody()` reconstructs the
accepted binding, before `subprocess.run()` dispatches the captured producer
bootstrap. Because the rehearsal deliberately precedes formal-root creation,
the formal namespace remained absent and uncontaminated.

## Why the prospective gates missed it

The real operator and formal-input paths accept the content-addressed
reproduction sidecar and verify it through its receipt. The standard-library
outer-launch unit fixture manually constructs an audit terminal with
`reproduction_runtime_file` set to the same stale
`audit-execution-02.runtime.json` literal used by the launcher. It therefore
tests internal agreement between two synthetic assumptions, not agreement with
the real `create_audit_reproduction_receipt()` producer.

The valid accepted-binding outer subprocess regression added for v1.16 used a
synthetic accepted audit package rather than one emitted by the real operator
and binding pipeline. Its audit package had five nonterminal members rather than
the real package's 15 and collapsed execution-capture and reproduction-sidecar
roles; its `audit-reproduction.json` was not a canonical
`prospect.wm001.audit-reproduction.v2` receipt. It exercised the repaired
large-file streaming and qualification-sidecar joins, but not this dynamic
audit-sidecar naming rule.
The operator-observed pre-root rehearsal exposed the remaining composition
mismatch before formal authority existed, and the retained source plus custody
metadata confirm that mismatch.
