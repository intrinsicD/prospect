## Limitations

1. This is a trusted-live-host receipt, not a portable or self-contained runtime.
   Kernel, CPU/microcode, loader, libc, systemd, filesystem, and owning UID remain
   outside the closure.
2. LCV authenticates the exact MM-007 files and independently replays only the surface
   consumed by a successor. It does not rerun or revalidate the historical scientific
   verifier chain.
3. The external execution receipt is a mutable, untracked `0664` supporting
   attestation outside the sealed 42-file package. Its artifact hashes and verifier
   outputs reproduce, but package bytes alone cannot prove the historical stdout,
   fsync syscall, or exact event sequence.
4. “Clean commit” refers only to the protocol's atomic namespace commit. It does not
   describe Git worktree cleanliness.
5. MM-011 has no authorization to score real targets yet. It requires a new task,
   frozen protocol/config/source binding, synthetic controls, independent pre-real
   audit, host revalidation, and copied LCV receipt before any one-shot real run.
